Page 1

Project overview

This project is a personal website for Sodbayar Battur, rebuilt to satisfy the final task requirements for a web technologies course. The project uses HTML, Tailwind CSS, a shared external CSS file, page-level style tags, JavaScript, and jQuery. The final result is a responsive multi-page site with separate sections for the homepage, personal story, image gallery, contact form, and documentation.

The design goal was to keep the original personality of the portfolio while restructuring it into a format that clearly demonstrates the required technical features. Instead of a single scrolling page, the project is now split into multiple HTML files so navigation and content structure are easier to understand and easier to grade.

Tailwind CSS remains the main styling method, because it allows quick layout work and consistent spacing. A small external stylesheet adds menu design, form states, table styling, link formatting, and reusable components that make the site easier to maintain. Page-level style tags are also used to satisfy the assignment requirement for embedded CSS.

Page 2

HTML structure and user interface

The website now contains five HTML files: index.html, story.html, gallery.html, contact.html, and documentation.html. This exceeds the minimum requirement of four HTML files. Each page shares a consistent header, navigation menu, footer, and visual language so the project feels like one system instead of disconnected pages.

The homepage introduces the purpose of the project and links to the other pages. The story page contains headings, paragraphs, an ordered list, and a bulleted list. The gallery page contains several images and a styled table that compares different chapters in life. The contact page contains a wide range of form controls including single-line text input, a multi-line text area, checkboxes, radio buttons, a color picker, a date selector, and buttons for submit and reset.

The user interface uses large headings, clearly separated sections, rounded content panels, and a warm neutral palette with orange accents. This keeps the site readable and modern while still looking like a personal project. Links are visibly styled and the active page is highlighted in the menu so visitors always know where they are.

Page 3

CSS usage

The assignment asked for CSS in multiple forms, and the project now includes all of them. Some formatting is written directly in a style attribute, for example the introductory label on the homepage. Page-level style tags define small custom sections such as gradients and print behavior. The external stylesheet styles/site.css contains shared rules for page backgrounds, navigation, links, table formatting, form controls, buttons, and content panels.

CSS selectors based on ID and class are both used. Examples include the #journey-table table area and reusable classes such as .nav-link, .form-input, .gallery-card, and .memory-item. The stylesheet also contains error states such as .is-invalid, which is applied by jQuery during form validation.

Tailwind handles the majority of spacing, layout, typography, and responsiveness. The external CSS file is intentionally compact and focused on the extra requirements that are cleaner to manage with named selectors. This keeps the project aligned with the user request to stay Tailwind-based while still meeting the assignment rubric.

Page 4

JavaScript and jQuery logic

JavaScript is used across the project for the mobile menu and for page interactions. jQuery is used on the gallery page and the contact page because the assignment explicitly requires JavaScript and jQuery. The gallery page uses jQuery to animate new content into the memory feed, modify an existing heading, and append a newly created HTML element after a button click.

The contact page demonstrates form checking in detail. Each field is validated when the form is submitted. If a field is incorrect, the script writes a specific error message into the matching error paragraph and adds a red frame by applying the .is-invalid class. This behavior covers single-line text input, textarea, radio button groups, checkboxes, and date selection. A successful submission opens a hidden feedback panel with animation and creates a new summary list of the submitted choices.

The script also demonstrates multiple selector styles: by tag name with $("input"), by name with selectors such as $("[name='focusArea']"), by class such as $(".form-input"), and by identifier with selectors such as $("#contact-form") and $("#feedback-panel"). This directly answers the rubric item about selecting elements in different ways.

Page 5

Conclusion and file summary

The finished project is now structured as a complete final submission rather than a single demonstration page. It includes multiple HTML files, embedded and external CSS, the required HTML elements, JavaScript and jQuery behaviors, and documentation that explains both the code and the user interface.

File summary: index.html presents the project, story.html contains personal background and lists, gallery.html contains images and a table, contact.html contains the validated form, documentation.html provides the written explanation, styles/site.css provides shared CSS, and scripts/site.js contains the mobile navigation, animation, and validation logic.

The project remains personal in tone, but it is now also much easier to inspect against the final task checklist. That balance was the main objective of the redesign.
